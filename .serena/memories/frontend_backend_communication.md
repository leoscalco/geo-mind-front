Backend integration is centralized in lib/api.ts. Base URL is process.env.NEXT_PUBLIC_GEOMIND_API_URL with fallback http://localhost:8010. There is no Next.js BFF, no app/api route handlers, and no server actions mediating backend calls. The frontend talks directly to FastAPI through fetch. app/page.tsx does server-side preload with getDashboardHoje() and getSnapshotFinanceiro(); client components then keep calling the same API functions in the browser. Main endpoints: GET /api/v1/dashboard/hoje, GET /api/v1/dashboard/{date}, POST /api/v1/dashboard/trigger, POST /api/v1/analisar, GET /api/v1/historico, GET /api/v1/financeiro/snapshot, POST /api/v1/simulacao/{noticiaId}, GET /api/v1/simulacao/{simulacaoId}/status, GET /api/v1/simulacao/{simulacaoId}/relatorio, GET /health. MiroFishPanel polls simulation status every 5s until concluded/falhou; NexusDashboard triggers dashboard generation then rechecks today’s dashboard after 3s.
